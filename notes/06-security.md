# 安全机制学习笔记

> OpenClaw 的安全设计：轻量但有效的安全框架。

## 安全设计理念

OpenClaw 采用 **相对轻量的安全框架**，相比商业产品限制更少，目的是便于实验和扩展。但仍提供必要的安全保障。

## 安全层次

### 1. 执行沙箱
- Tool 执行可选沙箱隔离
- 限制文件系统访问范围
- 网络访问控制

### 2. 权限系统
- Tool/Skill 级别的权限管理
- 敏感操作需要用户确认
- 分级权限模型

### 3. Session 隔离
- 不同 session 间状态隔离
- 防止跨对话数据泄漏

### 4. 本地优先
- 数据不离开本地机器
- 无云端数据存储（除 LLM API 调用外）
- 用户完全控制数据

## 安全与开放的平衡

| 方面 | OpenClaw 的选择 | 原因 |
|------|----------------|------|
| Shell 执行 | 允许（可配置限制） | 自主执行的核心能力 |
| 文件访问 | 允许（可配置范围） | 实用性优先 |
| 网络请求 | 允许 | Agent 需要联网能力 |
| 安全审核 | 轻量 | 开发者自行负责 |
