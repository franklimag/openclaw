# Memory 系统学习笔记

> OpenClaw 的记忆系统采用 Markdown-first 设计，透明、可编辑、可版本控制。

## 记忆文件体系

| 文件 | 用途 | 生命周期 |
|------|------|----------|
| `SOUL.md` | 人格定义、边界、语气 | 持久 |
| `AGENTS.md` | 操作指令 + 工作记忆 | 持久（动态更新） |
| `TOOLS.md` | 用户维护的工具使用说明 | 持久 |
| `BOOTSTRAP.md` | 首次运行引导脚本 | 一次性（完成后删除） |
| `IDENTITY.md` | 身份定义 | 持久 |

## 记忆类型

### 1. 滚动记忆（Rolling Memory）
- 对话 transcript
- 随着对话进行自动增长
- 受上下文窗口限制
- 通过 compaction 压缩

### 2. 持久记忆（Durable Memory）
- Markdown 笔记文件
- 手动或自动写入
- 跨 session 持久化
- 可以被用户直接编辑

### 3. 索引记忆
- SQLite 索引
- 加速语义检索
- 后端存储，用户无需直接操作

## 设计原则

1. **透明性** — 纯文本 Markdown，用户可随时查看和编辑
2. **可审计** — 所有记忆变更可追踪
3. **版本控制** — 可以用 git 管理记忆文件
4. **非不透明** — 拒绝黑盒式二进制存储

## 高级特性

### Activation/Decay 系统
- 记忆有"活跃度"概念
- 频繁使用的记忆更容易被召回
- 长期不用的记忆衰减优先级

### Domain RAG
- 领域特定的检索增强生成
- 结合知识图谱（3K+ facts）
- 语义搜索（多语言，GPU 7ms 延迟）
